IoT: Android Things Integration on the MIT App Inventor
By Thilanka Munasinghe

This is the final work product submission of the my Google Summer of Code 2017 project on Internet of Things listed on the appinventor-sources wiki. Specifically, I incorporated the Android Things SDK to MIT App Inventor Platform.

Android Things was unveiled fairly recently starting from Developer Preview 1 in December 2016. As of August 2017, the project is still evolving with the Developer Preview 5 being just released! Thus, application development utilizing Android Things has just begun by experienced developers, but using the the SDK is not yet simple enough for a typical MIT App Inventor user such as a high school student.

There are many famously known starter kits (Adafruit Project Kit, Sparkfun Project Kit) as featured in the official Google Android Things developer kits site, a direct integration of the Android Things opens the door for large number of school students, novel experimentalists, enthusiasts, and hobbyists to be able to do IoT projects along with App Inventor. Therefore, the goal of this Summer of Code 2017 project was to investigate ways to integrate the Google Things API features into MIT App Inventor, and contribute an MIT App Inventor extension component supporting Android Things to build IoT Android apps to the vast number of people worldwide.

Android Things also support different boards including Raspberry Pi, Intel Edison, Intel Joule, and NXP Pico via a common interface. Thus, this integration supersedes the capabilities offered by previous Arduino and Raspberry Pi integrations on MIT App Inventor.
